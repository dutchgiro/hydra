What is Hydra?

Hydra is multi-cloud application discovery, management and balancing service.

Hydra attempts to ease the routing and balancing burden from servers and delegate it on the client (browser, mobile app, etc).

Goal

Set up a Cloud Computing Service (IaaS) running onto more than one Cloud Provider in a federated way to be consumed as a single service from the consumer point of view.

It should review the different topologies, data distribution, referential data consistency, process distribution, load balancing, recovery policies, accounting gathering, write-once-run-everywhere (or sort of), and several new problems yet to rise.

Overall Architecture

Hydra must handle just the notification of the appropriate back-end server to supply a service for an identified or anonymous client. This notification must happen prior to the first request and, eventually, periodically/event-driven to have a refresh of the server assignment.

Once Hydra notifies the most suitable server it gets out of the service-consumer service-provider communication.

Therefore, Hydra offers a two sides operating interface: to the service consumer and to the provider server manager (the one who knows the actual server disponibility and workload). Those both interfaces can evolve to a publish subscription model.
